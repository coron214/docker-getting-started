# Docker Getting Started 学習ロードマップ

このロードマップは `docker-getting-started` の内容に沿って、Docker 初学者が順番に学べるようにしたものです。

## 使い方
- 各ステップを上から順に進める
- できたらチェックを付ける
- コマンドを実際に打って、結果を自分の言葉で説明できるか確認する

---

## Backlog

### 1. 最初のコンテナを動かす
- [ ] Docker Desktop をインストールする
- [ ] `docker run -d -p 80:80 docker/getting-started` を実行する
- [ ] ブラウザで `http://localhost` を開く
- [ ] `docker ps` で起動中コンテナを確認する
- [ ] `-d` と `-p 80:80` の意味を説明できる

**ゴール**
- コンテナを起動できる
- ポート公開の意味が分かる

### 2. コンテナとイメージの違いを理解する
- [ ] イメージとコンテナの違いを調べてメモする
- [ ] `docker images` を実行して一覧を見る
- [ ] `docker ps -a` を実行して停止済みも確認する
- [ ] 不要なコンテナを削除してみる

**ゴール**
- イメージは設計図、コンテナは実行中インスタンスだと説明できる

### 3. Dockerfile でアプリをビルドする
- [ ] Dockerfile の役割を説明する
- [ ] このリポジトリの Dockerfile を読む
- [ ] `docker build -t getting-started .` を実行する
- [ ] 作成したイメージからコンテナを起動する

**ゴール**
- Dockerfile からイメージを作る流れが分かる

### 4. 起動・停止・削除を一通りやる
- [ ] 起動中コンテナを停止する
- [ ] 停止したコンテナを削除する
- [ ] 名前付きでコンテナを起動する
- [ ] ログを確認する

**ゴール**
- コンテナ管理の基本コマンドに慣れる

### 5. Volumes でデータを永続化する
- [ ] volume の意味を説明する
- [ ] volume を使わない場合と使う場合の違いを試す
- [ ] コンテナを削除してもデータが残ることを確認する

**ゴール**
- コンテナを消しても残したいデータをどう扱うか分かる

### 6. Bind Mount で開発しやすくする
- [ ] bind mount の意味を説明する
- [ ] ローカルのファイル変更がコンテナに反映されることを確認する
- [ ] volume との違いをまとめる

**ゴール**
- 開発用マウントと永続化用マウントの違いが分かる

### 7. コンテナ同士を通信させる
- [ ] Docker ネットワークの役割を調べる
- [ ] 複数コンテナ構成を試す
- [ ] アプリと別サービスが通信している流れを図にする

**ゴール**
- 単一コンテナだけでなく連携の考え方が分かる

### 8. Docker Compose を使う
- [ ] `docker-compose.yml` または `compose.yml` を読む
- [ ] `docker compose up` を実行する
- [ ] サービス定義を読んで、何が自動化されているか説明する
- [ ] `docker compose down` を実行する

**ゴール**
- 複数コンテナ構成を設定ファイルで扱える

### 9. レイヤーキャッシュを理解する
- [ ] Docker build のレイヤーの考え方を調べる
- [ ] Dockerfile の命令順でビルド時間が変わる理由を理解する
- [ ] キャッシュが効く例・効かない例を1つずつ挙げる

**ゴール**
- ビルド高速化の基本が分かる

### 10. Multi-stage build を理解する
- [ ] multi-stage build の目的を説明する
- [ ] build 用と runtime 用の分離を説明する
- [ ] イメージサイズが軽くなる理由を説明する

**ゴール**
- 本番向けの Dockerfile 改善の基本が分かる

---

## Done の基準
- コマンドをコピペだけでなく意味付きで説明できる
- 自分で1回エラーを解消した経験がある
- 小さなアプリを1つ Docker 化できる

## 最終課題
- [ ] 自分のアプリを1つ Docker 化する
- [ ] `Dockerfile` を作る
- [ ] `docker compose.yml` を作る
- [ ] データ永続化が必要なら volume を使う
- [ ] README に起動方法を書く

## 追加メモ
- 分からないコマンドは、実行前に「何をするコマンドか」を1行で書く
- 学習ログを issue や README に残すと復習しやすい
